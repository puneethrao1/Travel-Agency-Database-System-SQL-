# 💾 Travel Agency Database System (SQL)

## 🔎 Overview

This project demonstrates the design and implementation of a **relational database system for a travel agency** using **MS SQL Server**.

The system manages customer records, travel packages, and booking information using structured relational tables and SQL queries.

The project showcases core database concepts including **database design, table relationships, data management operations, and SQL query execution.**

---

## 🛠 Technologies

- Microsoft SQL Server
- SQL
- Relational Database Design
- Data Management Queries

---

## ✨ Features

### 🗄 Database Design
Created a relational database named **TravelCompany** with structured tables.

### 👤 Customer Management
Stores customer information including names, contact details, and identification fields.

### 🌍 Travel Packages
Maintains travel package data including destination, pricing, and descriptions.

### 📑 Booking Records
Tracks customer bookings associated with travel packages.

---

## ⚙️ The Process

### 1️⃣ Database Creation
Created the **TravelCompany** database and defined relational tables including:

- Customers
- Packages
- Bookings

Primary keys and appropriate data types were defined to maintain data integrity.

---

### 2️⃣ Data Insertion
Inserted sample records into the database using SQL **INSERT INTO** statements to simulate real-world travel booking data.

---

### 3️⃣ Data Management

Implemented core SQL operations including:

- **SELECT** queries to retrieve customer and package data
- **UPDATE** statements to modify existing records
- **DELETE** statements for record management
- **WHERE** clauses for filtered queries
